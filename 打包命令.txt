内容保护提示词

请为当前 Flutter 项目增加基础内容保护：
1. 不改变现有页面表现
2. 将课程、题目、错误、动画步骤等内容从代码中抽离到 assets 数据文件
3. 设计 ContentRepository 统一读取内容
4. 预留后续加密/压缩接口
5. 保持现有测试通过



打包命令

flutter build apk --release --obfuscate --split-debug-info=build/symbols











1. 哪些内容要抽离

建议抽离这些内容：


首页菜单：标题、副标题、图标 key、颜色、路由、排序。

课程集合配置：例如 BFS 基础、迷宫 BFS，每组课程的 id/title/assetPath/order。

题库集合配置：题库 id、标题、题目路径、总题数、及格线、过滤策略。

错误集合配置：错题/误区路径、分类、难度。

动画演示配置：场景 id、标题、步骤文件、网格行列、播放速度、是否教师模式可用。

页面 UI 文案：加载失败、暂无内容、按钮、筛选项、结果页文案。

进度统计规则：哪个 quizSet 对应哪个 score key，目前 bfs_basic、20 这些是写死在页面里的。


2. 新增哪些 model

建议新增：


ContentManifest：整个内容目录入口，记录版本、默认专题、所有模块。

LearningTopic：一个专题，例如 bfs，以后可以扩展 dfs、dp。

ContentModule：首页模块项，例如 lesson、quiz、animation、mistake。

LessonSet：课程集合，包含 id/title/subtitle/assetPath/order。

QuizSet：题库集合，包含 id/title/assetPath/passScore/totalScore。

MistakeSet：错题集合，包含 id/title/assetPath/category。

AnimationScenario：动画场景，包含 id/title/assetPath/rows/cols/playIntervalMs。

AppCopy 或 UiTextBundle：统一存页面文案。

ContentRef：轻量引用，用于页面通过 id 找内容，而不是硬编码路径。


现有 Lesson、Quiz、Mistake、BfsStep 可以保留，作为“内容条目模型”。

3. 新增哪些 repository

建议新增 lib/repositories/：


JsonAssetRepository

统一封装 rootBundle.loadString + json.decode，替代页面和 service 里重复读 JSON。

ContentManifestRepository

读取 assets/data/content_manifest.json，提供专题、模块、路径索引。

LessonRepository

根据 LessonSet.id 或 assetPath 加载课程列表。

QuizRepository

根据 QuizSet.id 加载题库，同时提供题目校验。

MistakeRepository

加载常见错误内容。

AnimationRepository

加载 AnimationScenario 对应的 BFS 步骤。

UiTextRepository

统一读取页面文案，后续支持中文/英文或修复乱码。

ProgressRepository

可以先包一层现有 progress_service.dart，让进度 key 逐步从写死值迁移到内容 id。


现有 services 可以先保留，repository 初期调用 service 或替代 service 都可以。为了低风险，推荐先“新增 repository，不删除旧 service”。

4. 哪些页面需要适配

优先级如下：

home_page.dart

首页菜单从 ContentManifest 读取，不再在页面里写死。
lesson_page.dart

当前写死 assets/data/lessons/bfs_basic.json，应改成根据 LessonSet 加载。
quiz_page.dart

当前写死题库路径、分数统计、过滤文案，应接入 QuizSet 和 ProgressRepository。
mistake_page.dart

当前写死 bfs_mistakes.json，应通过 MistakeSet 加载。
animation_page.dart 和 teacher_mode_page.dart

当前页面内直接 rootBundle.loadString('assets/data/bfs_steps.json')，应改走 AnimationRepository。
progress_page.dart

当前写死 bfs_basic、20 分等规则，应从 QuizSet/manifest 获取。
app/router.dart

短期可不动；中期支持 route arguments，例如打开不同 lessonSet/quizSet。
settings_page.dart

可后置，只需要读取 UiTextBundle 和内容版本信息。

5. 如何保证现有功能不变

核心策略：先加新层，不替换旧行为；用默认 manifest 完全映射现有文件。

建议建立一个默认 manifest，大概表达：

json



{
  "version": 1,
  "defaultTopicId": "bfs",
  "topics": [
    {
      "id": "bfs",
      "lessonSets": [
        {
          "id": "bfs_basic",
          "assetPath": "assets/data/lessons/bfs_basic.json"
        },
        {
          "id": "bfs_maze",
          "assetPath": "assets/data/lessons/bfs_maze.json"
        }
      ],
      "quizSets": [
        {
          "id": "bfs_basic",
          "assetPath": "assets/data/quizzes/bfs_quiz.json",
          "totalScore": 20
        }
      ],
      "mistakeSets": [
        {
          "id": "bfs_mistakes",
          "assetPath": "assets/data/mistakes/bfs_mistakes.json"
        }
      ],
      "animationScenarios": [
        {
          "id": "bfs_maze_steps",
          "assetPath": "assets/data/bfs_steps.json",
          "rows": 5,
          "cols": 5
        }
      ]
    }
  ]
}



这样页面迁移后读到的仍是同一批 JSON，用户看到的功能路径不变。

保证方式：


每迁移一个页面，先保持相同 asset path、相同排序、相同 UI。

保留原 provider，例如 lessonsProvider(String assetPath)，新增 lessonSetProvider(String id)。

原测试继续跑，再补 repository 测试。

进度 key 不立刻改名，继续兼容 bfs_basic。

Progress.fromJson 保留旧字段，新增字段只做可选。

repository 失败时返回明确异常，让页面现有 error UI 继续生效。


6. 分步骤改造计划


内容盘点与 manifest 引入

新增 content_manifest.json，只描述当前已有内容，不改变页面。



新增基础 model

加 ContentManifest、LearningTopic、LessonSet、QuizSet、MistakeSet、AnimationScenario。



新增 repository 层

先实现 JsonAssetRepository 和 ContentManifestRepository，再实现 lesson/quiz/mistake/animation repositories。



新增 provider，不替换旧 provider

例如保留 lessonsProvider(assetPath)，新增 lessonSetProvider(setId)。



先迁移 Animation/Teacher

这两个页面现在直接读 bundle，收益最大。迁移后仍读取 assets/data/bfs_steps.json。



迁移 Lesson/Mistake/Quiz

从硬编码 asset path 改为通过 set id 读取。先默认打开 bfs_basic，不改导航体验。



迁移 Home 和 Progress

首页从 manifest 生成菜单；进度页从 QuizSet.totalScore 获取总分，不再写死 20。



抽离 UI 文案

建 ui_text_zh.json 或 Dart 常量文件，统一修复页面乱码。这个可以独立做，但建议在内容系统稳定后做。



补测试

增加 manifest 解析测试、repository 加载测试、页面默认内容 smoke test、进度兼容测试。



最后清理旧 service 或保留适配层

如果 repository 已完全覆盖，再考虑把 services 改名为 data source，或逐步删除重复代码。



整体方向是：把页面从“知道 JSON 文件在哪里”改成“只知道自己要展示哪个内容 id”。这样以后加 DFS、图论、动态规划，基本就是加 JSON 和 manifest，不需要复制一批页面。
